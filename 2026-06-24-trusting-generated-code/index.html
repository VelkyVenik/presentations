Žádný typ není „lepší" — každý má své místo. Dnes mluvíme o agentním vývoji.

Cíl: nemuset psát ani číst kód.
Lidé popíšou cíl, schválí plán a zhodnotí výsledek. Agenti obstarají všechno mezi tím. Nezbavujeme se kontroly — děláme ji rychleji a líp.
Když kód nikdo nečte — proč mu věřit?
Odpověď není ve čtení řádků. Je v procesu.
01Vibe coding

Prototype

Jde o pochopení, ne o kód

  • Kód stejně zahodím — pod kapotou je mi to jedno
  • Screenshot jako reference, popíšu use-casy a jedu (Claude Code)
  • Soustředím se na vzhled a interakci
  • Ukážu zadavateli, prodiskutujeme, ladím
02Agentic

Implementation

Standardizovaný stack

  • Skills řeknou agentovi pravidla — stack, styly, CI/CD
  • Sdružené v našem Claude Code Marketplace
  • Spec + plán → agenti staví (plán = největší přidaná hodnota)
  • Ladím, dokud nejsem spokojený technicky i funkčně
03Kontrola

Review

Nezávislí agenti prověří zbytek

  • Spustí se po každém PR — kód nikdo nečetl
  • CodeRabbit (review), Wiz (bezpečnost), AISLE
  • Fixy navrhne agent (Codex)
  • Teprve pak člověk — tady se rodí důvěra
04Zisk

Operation

Nasaď, sleduj — a sklízej

  • Aplikace je živá, jdou na ni reální uživatelé
  • Telemetrie (PostHog) ukáže, co se reálně děje
  • Zpětná vazba → další iterace
  • Tohle je zisk — všechno předtím byla příprava
